VS Code 设置打开文件时自动猜测字符集编码

发表于 2023-11-06 更新于 2024-10-20 阅读次数：
如题，作为备忘。

省流，VS Code 是默认设置打开文件时不自动猜测字符集编码的，他一般默认按 UTF-8 编码来打开文本文件。如需勾选，将设置中Files: Auto Guess Encoding，该项勾选即可。如要搜索该项，其他可搜索的中文关键词包括：字符集，编码。或者搜索或设置settings.json相应的配置项"files.autoGuessEncoding": true。

主要想发发牢骚，个人认为他 VS Code 没默认打开该选项是一个错误。众所周知，现在的文本文件的编码一般是设置为 UTF-8，或者说一般要设置为 UTF-8，这样才合适而且通用。VS Code 当然也是如此，对于文本文件的默认打开和保存方式都是 UTF-8，并且在右下角可以灵活修改设置，这个没什么问题。

然而在 Visual Studio 编写的代码文件，默认是 GBK 或者说 GB2312 来编码中文字符。当然了这个可以在设置里修改，或者是安装相关插件后，同样方便在编辑器的右下角修改。然而对于 masm32 编程，涉及中文字符串的时候，好像还真得只能用 GB2312 编码，没法用 UTF-8 编码。

很多时候我需要用 VS Code 打开 Visual Studio 项目里的代码内容。有个问题在于，每次点开那个代码文件，VS Code 都是默认以 UTF-8 打开，还得我手动点击右下角修改打开方式。我关掉这个文件之后，下次点开的话，他又给我用 UTF-8 打开了，每次都得重新搞。但有时候我会忘了，代码改着改着就点击保存了，一看发现已经乱码了，这个时候我再点右下角，反复捣腾“通过编码重新打开”，“通过编码保存”，有时候就把自己绕进去了，而且还不方便撤回。
