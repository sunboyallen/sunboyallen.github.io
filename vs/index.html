项目级别的C++开发，特别是要用第三方库的，还得是Visual Studio

发表于 2024-03-12 阅读次数：
给个暴论，特别是那种要用第三方外部库的，非Visual Studio不可，另外搭配CMake食用更好。

VS Code的不足

我之前曾写过一篇博客关于“如何在VS Code给C++添加OpenCV库”，以及遗留的问题，主要是依然没做到静态编译，也就是exe没法单独跑，得加各种dll文件。以前一直嫌弃VS笨重，然而当我最终还是在VS上配置并且实现了静态编译之后，我就把那篇博客删掉了，没必要了。

像这种项目级别的C++开发，特别是要用第三方库的，不用Visual Studio是没意义的。如果你只是想在自己的电脑上简单玩玩OpenCV，随便搞搞图片，那其实VS Code也够用。CMake先编译，配完加环境变量，跑还是能跑的，无非是如果exe单独跑的话还得加各种杂七杂八的dll动态库文件。但如果是项目级的，这样显然是不够的，如果需要exe能单独跑呢，之前那样搞，放到别人家电脑里，指不定对方还缺少什么dll呢，这也暴露了VS Code在面对C++第三方库的时候一直难以解决的静态编译问题。或许有些第三方库在VS Code的环境下有些歪门邪道，奇技淫巧（没有贬低的意思），但显然不是通用的方法。

CMake

好就好在现在的第三方库的编译基本都有提供CMake的方式。使用cmake-gui，将CMakelists.txt文件所在的文件夹作为source code，当然直接把他拖进去更方便。关键是Configure的时候要搭配Visual Studio，选择编译器时，选择自己电脑相应的VS的版本，生成之后点击Open Project，就会自动打开VS，然后在里面install生成了。当然这里有些建议就是，打开完VS可以关掉，再以管理员身份打开VS，打开之前那个项目来生成，不然可能由于权限不够，生成失败。

当然这里有些第三方库会好心地给你提供直接在Visual Studio打开进行编译生成的方式，不过我依然不建议用这种方式，还是按照之前的，通用的方式。我试过了，之后在Visual Studio用于编译自己写的C++代码的时候会出问题。包括他们提供好的现成的编译好的带有lib库之类的，也不一定好用，还是自行在VS里生成比较好。

Visual Studio

之后的操作就很常规了，项目属性一通配置，网上一找一大把，我看基本够用了，不多说。我只是自己补充一些。比如，项目级开发我是比较推荐把第三方库直接拷贝一份在项目目录里的，项目属性引入外部库，链接之类的，也直接用相对路径，比如$(ProjectDir)lib之类的写法，这样跨端开发能省去很多重复的操作。

我自己试过了不少第三方库了，清一色基本是通用的操作流程：CMake编译，VS生成，VS项目配置。我简单列出一些。

OpenCV

https://opencv.org/

https://github.com/opencv/opencv

下载源代码，根目录就是。网上配置的教程比较多了，主要是这篇：https://blog.csdn.net/bby1987/article/details/108859363，详细说了Configure时候如果遇到下载超时的处理。手动下载，放到指定目录，有的还要MD5校验，加上MD5值。

LZ4

https://lz4.org/

https://github.com/lz4/lz4

直接clone过来，在build/cmake里。

zlib

https://zlib.net/

https://github.com/madler/zlib

可以官网点击下载zip，根目录就是了。
