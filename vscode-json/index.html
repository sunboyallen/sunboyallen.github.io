VS Code的配置文件

发表于 2023-06-02 更新于 2024-01-06 阅读次数：
主要是指VS Code里的json格式配置文件。官方文档有很详细的说明了，具体内容我就不照搬，只是自己做一些补充。

tasks.json

https://code.visualstudio.com/docs/editor/tasks

一个常用的应用场景是用 MinGW 来编译 C++程序。

https://code.visualstudio.com/docs/cpp/config-mingw

官方文档中给出了快捷生成配置模板的方式，让 VS Code 帮你写好一个。

https://code.visualstudio.com/docs/editor/variables-reference

任务中，至少要有一个标注为build种类的任务。具体做法，就是像上面文件那样，设置group属性。这样，配置完任务后，如果要运行任务，则在菜单栏“终端”选择“运行任务”，选择特定的任务。或者直接选择“运行生成任务”，就是直接运行build种类的任务。

对于需要先后执行多个任务的场景，一般的做法是，如果执行任务 A 之前需要先执行任务 B，那么在任务 A 里设置dependsOn属性，其值为数组，数组中的元素就是依赖的任务名字，即相应的label值。如果依赖的任务本身还有其要依赖的任务，那么类似地去填写那个任务的dependsOn属性，以此类推即可。

launch.json

https://code.visualstudio.com/docs/editor/debugging

在launch.json文件的右下角，有“添加配置”按钮，可以根据需要来选择配置。

以 C++程序调试为例，点击右下角“添加配置”按钮，选择 C/C++: (gdb) 启动，就会添加一个配置。

需要修改的参数大致如下。

program表示要调试的程序，其值为程序的绝对路径。这里一般是改为${fileDirname}\\${fileBasenameNoExtension}.exe。

miDebuggerPath修改为自己 gdb.exe 的绝对路径，比如我的是D:\\mingw64\\bin\\gdb.exe。

此外，一般需要添加preLaunchTask属性，用于在执行任务之后再进行调试，其值即为 tasks.json 中相应的任务名称。比如，这里的例子中，就是C/C++: g++.exe 生成活动文件。

settings.json

https://code.visualstudio.com/docs/getstarted/settings

如上所说，包括两种设置，一种是用户设置，一种是工作区设置。前者是全局的，后者只针对当前工作区。

https://code.visualstudio.com/docs/getstarted/settings

这里说了，settings.json里的设置，和 VS Code 里的设置选项是一一对应的。默认的设置不会在settings.json里体现，只有修改了默认设置的值，才会在settings.json里显示出。
